Objective

The objectives of the project are as follows:

  • Survey of statistical/probabilistic algorithms based on HMM and Bayes for behavioural network anomaly detection.
  • Drawing a comparison between HMM and Bayes models on different parameters involving working, detection capabilities, limitations etc.
  • Implementation of an Adaptive NIDS using HMM and Bayes model.
  • Evaluation of algorithms on DARPA datasets involving parameters such as false positives and false negatives.
  • Comparison of results of the algorithm evaluation.

Scope of the Project


The scope of this project is to survey the openly available machine learning algorithms available in the space of behavioral network anomaly detection and carry out the comparative study. The idea is to understand the current state-of-the-art algorithms in this space and evaluate them on standard benchmark datasets.

Background


  • The current network security technologies (such as Firewall, IDS/IPS) do provide security to networks but have limitations in terms of detecting only the known attack patterns
  • There are many unknown vulnerabilities in the network protocols, OS and software which can be potentially exploited by the hackers/attackers
  • The system becomes prone to attacks until the signatures of such attacks are found and configured in the signature databases of network security solutions
  • The moment attack pattern changes, the signature based solutions fail

Introduction


  • An Intrusion can be termed as any activity which is aimed at disrupting or denying a service by gaining unauthorized access
  • Various attacks like DDoS, Identity spoofing, Ping of Death, Buffer Overflow etc are used to intrude and disrupt a network
  • Intrusion Detection Systems are used to prevent such intrusions by detecting them at an earlier stage
  • All Intrusion Detection Systems use one of two detection techniques:
    • Signature Based Detection
    • Anomaly Based Detection

Motivation


  • The behavioral network anomaly detection is another paradigm which monitors the network traffic and creates a normal profile representing normal network behavior
  • The moment any deviation in the network behavior is found which drastically deviates from the normal behavior, the alarm is raised
  • The basic idea of this project is to investigate the machine learning based network anomaly detection algorithms using behavioral paradigm that can ensure maximum detection rate and minimal false alarm rate

Existing System

Drawbacks of the Existing System

  • Signature based systems can only detect an intrusion attempt if it matches a pattern that is in the database, therefore causing databases to constantly be updated
  • When ever a new virus or attack is identified it can take vendors anywhere from a few hours to a few days to update their signature databases
  • Hosts that are subjected to large amounts of traffic, the IDS can have a difficult time inspecting every single packet that it comes in contact, which then forces some packets to be dropped leaving the potential for hazardous packets getting by without detection
  • Systems can suffer a substantial performance slow down if not properly equipped with the necessary hardware to keep up with the demands

Proposed System

Advantages

  • New threats can be detected with out having to worry about database being up to date
  • Very little maintenance once system is installed, it continues to learn about network activity and continues to build its profiles.
  • The longer the system is in use the more accurate it can become at identifying threats
